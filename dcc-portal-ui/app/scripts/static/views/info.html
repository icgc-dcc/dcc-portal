Access Raw Data

Public Repositories Hosting ICGC Data

ICGC member projects are required by ICGC policies to submit their raw sequencing reads, and other primary data, to controlled access public repositories. The official ICGC DCC-supported repository for ICGC sequencing reads is the European Bioinformatics Institute's (EBI) European Genome-phenome Archive (EGA).

For the ICGC member projects which are part of the U.S TCGA project raw data will be hosted either at the TCGA DCC or the Cancer Genomics Hub (CGHub).

Applying for Access to Controlled-Access Datasets

Access to the ICGC sequencing datasets held at the EGA is granted by applying through ICGC's Data Access and Compliance Office (DACO).

Overview of Available Raw Data

All controlled-access ICGC datasets held at EGA are associated with the ICGC DACO's EGA DAC accession #: EGAC00001000010.

Methods

Gene model

The ICGC Data Portal uses gene models produced by Ensembl (http://www.ensembl.org).

The canonical gene data (release 69) is available from Ensembl FTP site at ftp://ftp.ensembl.org/pub/release-69/mysql/homo_sapiens_core_69_37. We use a further-processed gene model file provided by snpEff which is available for download from http://sourceforge.net/projects/snpeff/files/databases/v3_3/snpEff_v3_3_GRCh37.69.zip/download. This file is then transformed into a document-oriented data store for integration with other data types, such as, somatic mutations.

Gene consequence annotation

As part of the effort to enable cross cancer projects functional analysis, we perform standardized gene consequences annotation, using Sequence Ontology's controlled vocabulary regarding mutation-induced changes (www.sequenceontology.org). This type of annotation provides a list of predicted gene effects, such as a non-conservative amino acid substitution, but does not attempt to prioritize the changes by their functional impact. Simple somatic mutation data submitted by member projects are reannotated using the variant annotation tool SnpEff (v3.3c, http://snpeff.sourceforge.net/) recommended by the ICGC Mutation Consequences and Pathways (MUCOPA) Working Subgroup (Nature Methods, 2013, PMID:23900255).

Simple somatic mutation frequencies

In order to assess how often a particular simple somatic mutation (SSM) occurs in cancer patients, we establishes specific criteria to determine whether SSMs reported from different donors are considered to be the same. When all of these fields from two reported SSMs: chromosome, chromosome start, chromosome end, mutation type, DNA change, genome assembly version are identical, they are merged into one mutation entity. With this, we build a non-redundant collection of all simple somatic mutations. Each mutation entry in the collection is assigned a stable identifier for persisted referencing portal wide and across releases. Information for linking these non-redundant SSMs to the reported ones is also kept, enabling mutation counts across donors.

Submitter Tools

This section provides instructions for ICGC projects on releasing data through the ICGC DCC and public repositories

Using the ICGC DCC Data Submission System (Beta)

The ICGC DCC now supports data submissions through a web-based submission portal: https://submissions.dcc.icgc.org.

The submission system is currently accepting files using ICGC DCC data specification, which is available here as an HTML document.

During the beta period, please refer to the ICGC Data Submission Bulletin, and submission system software documentation for instructions and important notes to assist you in using this system to release your project's data through the DCC data portal.

Please contact ICGC Data Coordination Centre if you have any questions or comments about the data submission process.

Virtual machine for local data validation

The ICGC DCC offers a standalone virtual machine that allows you to locally validate ICGC submissions before uploading complete datasets to the online web-based data submission system. Please note that, as opposed to the DCC online submission system which is running on a computer cluster, this VM can be deployed only on a single server. It is not therefore suitable for validating entire datasets but rather meant for testing small subsets of data before submission to DCC. To use this virtual machine, please download the VirtualBox package from here . Instructions on how to use the VM can be found here .
